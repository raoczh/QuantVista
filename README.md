# QuantVista

[![License](https://img.shields.io/badge/License-Apache_2.0-blue.svg)](LICENSE)
![Go](https://img.shields.io/badge/Go-1.25-00ADD8?logo=go&logoColor=white)
![Vue](https://img.shields.io/badge/Vue-3-4FC08D?logo=vuedotjs&logoColor=white)

QuantVista 是一个**个人自用的 AI 股票研究平台**：围绕 A 股行情数据、新闻情绪、AI 分析与推荐、持仓与模拟盘，形成可复盘、可追踪、可配置的一站式研究工作台。

系统不输出「买入答案」，而是输出**研究观察与交易计划**——给出数据依据、逻辑、风险与跟踪条件，每个 AI 结论都绑定数据快照、经过程序化证据核验，决策由用户自行承担。

## ✨ 功能特性

**行情与数据**

- 📈 **市场看板**：大盘指数、涨幅/成交额榜、板块涨跌、涨跌家数情绪、两市资金流向、AI 今日观点
- 🔍 **个股详情**：K 线（前复权，MACD/BOLL/均线）、筹码峰分布、主力资金流、龙虎榜、财务摘要与 F10、公告列表、机构观点（研报评级分布/目标价偏离/机构调研密度）、技术面五维评分
- 🗺️ **板块热力图**：行业/概念板块热度全景与板块详情页（指数 K 线、成分股联动、板块主力资金历史、行业估值中位数与分位）
- 📰 **新闻情绪**：7×24 快讯采集（间隔可配 1~120 分钟）+ LLM 情绪增强（个股关联、利好利空、当日聚合情绪分，可总闸关闭走纯规则）

**AI 研究**

- 🤖 **AI 分析**：五模块（个股/全市场/板块/自选/持仓）精确提示词、多角色 panel（技术/动量/风险/反方）、变化检测、回溯诊断（as-of 历史时点复盘）、个股自动附**交易计划**（买点/止盈/止损/仓位）
- 💬 **个股问答**：流式回答、多轮追问复用数据快照、风险标志程序化解析
- ⚖️ **横向对比**：多标的估值/评分/技术面对照 + AI 点评
- 🎯 **短线/长线推荐**：四阶段流水线——多源建池 → 用户硬过滤（股价区间、排除创业板/科创板等，可自定义）→ 本地量化评分（零 LLM 成本）→ LLM 仅对 Top 名单精选/否决
- ⚡ **生成链路异步化**：推荐/日报生成接口秒回 + 前端轮询，后台任务不受浏览器/反代超时影响；AI 上游超时自动**量化降级**（ATR 规则价位、透明标记、置信度恒低）
- 🛡️ **信任层（全 AI 链路）**：程序化证据核验（引用数字与快照逐一比对）、程序合成置信度、可选 AI 复核员、数据快照透明面板；全部 AI 产出标注所用 LLM 配置与模型
- 🧪 **策略选股**：21 个内置白话策略（短线/波段/中线，基于 52 因子宽表）+ 自定义条件树 + **AI 白话建策略**（自然语言解析为条件树，无法映射的表述如实提示不硬凑）
- ⏪ **回测时光机**：策略在历史时点的选股结果与后续走势验证（次日开盘成交/一字板跳过/涨跌停顺延等 A 股真实约束）

**跟踪与交易**

- 📌 **推荐追踪**：止盈/止损盘中触达判定、相对上证的超额收益 alpha、7/14/30 日节点收益、胜率统计、复盘待办与已读消项
- 📋 **收盘日报**：交易日自动生成今日复盘 + 明日短线计划，自动挂卖点提醒，可推送 Server酱/Webhook，支持删除重生成
- ⏰ **提醒与待办**：到价/异动/均线/突破/放量条件提醒，今日待办聚合提醒命中、复盘信号、逻辑卡到期
- 💼 **持仓管理**：买入成本含费税的实时盈亏、平仓结构化复盘、组合风控信号、推荐↔持仓血缘
- 🎮 **模拟交易**：虚拟 10 万本金、真实行情成交与费税；指数 ETF 交易（免印花税费率）
- 🗂️ **研究辅助**：投资逻辑卡（失效条件+一键体检）、投资笔记、自选分组与研究阶段漏斗

**系统与管理**

- ⚙️ **LLM 配置**：多配置管理、OpenAI 兼容（`/v1/chat/completions` 与 `/v1/responses` 双端点）、测试连接、每用户次数配额熔断
- 🔄 **LLM 回退**：未配置 LLM 的用户可自动回退管理员指定配置（总闸+指定配置可控），后台任务共用系统默认 LLM
- 🖥️ **管理后台**：用户启停与配额、**LLM 调用审计**（全用户请求/响应明细，90 天自动清理）、新闻采集配置、GitHub 凭证与注册开关、数据源同步日志
- 🎨 **6 套主题**（亮/暗 × 多色系）+ 移动端适配

## 🚀 快速开始

### 源码运行（开发）

要求：Go ≥ 1.25，Node.js ≥ 20（实测 v24）。

```bash
# 后端（SQL_DSN=local 使用本地 SQLite，零外部依赖）
cd server
SQL_DSN=local go run .

# 前端（另开终端）
cd web
npm install
npm run dev
```

访问 `http://localhost:5173`，首次启动会引导创建管理员账号。

### Docker 部署（生产）

```bash
docker buildx build --platform linux/amd64 -t <你的镜像名>:tag --load .
docker push <你的镜像名>:tag
```

生产环境用 `deploy/docker-compose.example.yml` 为模板编排（MySQL + Redis + 应用），密钥经 `deploy/.env` 注入。完整流程（宝塔编排、反代与真实 IP、数据库自动迁移、备份恢复）见 [部署说明](docs/DEPLOYMENT.md)。

## ⚙️ 环境变量

| 变量 | 必填 | 说明 |
| --- | --- | --- |
| `SQL_DSN` | 是 | `local`/留空 = 本地 SQLite（开发）；MySQL DSN = 生产模式（触发密钥强校验） |
| `SESSION_SECRET` | 生产必填 | JWT 签名密钥，`openssl rand -base64 36` 生成；生产环境弱值拒绝启动 |
| `ENCRYPTION_KEY` | 生产必填 | 敏感字段加密密钥（LLM API Key 等）；**备份恢复须用同一密钥** |
| `PORT` | 否 | 监听端口，默认 `3000` |
| `SQLITE_PATH` | 否 | SQLite 文件路径（默认程序目录） |
| `REDIS_CONN_STRING` | 否 | Redis 连接串，缺省不启用缓存 |
| `ALLOWED_ORIGINS` | 否 | 生产 CORS 白名单，逗号分隔 |
| `TRUSTED_PROXIES` | 否 | 反代场景可信代理地址（如 `172.18.0.1`），不设则不信任代理头 |
| `GITHUB_CLIENT_ID` / `GITHUB_CLIENT_SECRET` | 否 | GitHub OAuth 首启种子，之后以管理后台配置为准 |
| `TUSHARE_TOKEN` | 否 | Tushare Pro token（分档按需启用） |
| `DEFAULT_MARKET` | 否 | 默认市场，当前仅 `cn` |
| `DEBUG` | 否 | 调试日志，默认 `false` |

## 🔐 安全特性

- **LLM API Key 加密落库**：AES-256-GCM 加密存储，模型字段 `json:"-"` 永不回显给前端；推送通道密钥、GitHub secret 同等处理
- **JWT 双 token**：短期 access + 轮换 refresh，`TokenVersion` 支持单用户强制全端登出（禁用用户即时生效）
- **SSRF 防护**：LLM Base URL 与 Webhook 仅放行公网 http/https，内网地址仅管理员（或管理员所有的回退配置）可用
- **认证限流**：登录/注册按 IP 限流，反代场景配 `TRUSTED_PROXIES` 后按真实客户端 IP
- **生产密钥 fail-fast**：连接真实 MySQL 时占位/弱密钥直接拒绝启动
- **调用审计**：全用户 LLM 调用明细（含请求/响应全文）仅管理员可见，90 天自动清理
- 备份恢复依赖同一 `ENCRYPTION_KEY`，密钥丢失则密文字段需重新录入

## ⚠️ 风险与免责

### 法律风险（重要）

在中国大陆，**向不特定对象推荐具体证券及买卖时点属于证券投资咨询业务，需证监会牌照**。无证「荐股」即使附带免责声明也可能违规。因此：

- 本项目定位为**个人自用研究工具**，谨慎对待公开多用户部署；公开前必须做合规评估。
- 全产品统一用「**研究观察 / 交易计划 / 候选标的**」表述，**禁止**出现「推荐买入」「稳赚」「目标收益」等确定性、诱导性措辞。
- 短线模块表述为「短线交易计划 / 买入观察计划」，强调条件与风险，不做主动买卖指令。

### 数据与模型风险

- 所有 AI 分析与推荐仅供研究参考，**不构成投资建议**。
- AI 分析可能出错，数据可能延迟或不完整；分析结论与输入数据快照绑定。
- 用户需自行承担投资决策风险。
