# QuantVista

QuantVista 是一个完整全栈的 AI 股票研究与交易辅助平台。项目目标不是给出不可解释的“买入答案”，而是围绕市场数据、用户自选、AI 分析、短线/长线推荐、已购入持仓和推荐后追踪，形成可复盘、可持久化、可配置的股票研究工作台。

## 项目定位

- **部署形态：个人自用研究工具**。当前阶段定位为单人/极少数人自用，不做公开多用户商业部署。这一定位直接决定了合规风险等级和措辞口径（见“风险与合规”）。如未来需要公开部署，必须先补充合规评估。
- 面向个人投资研究场景，帮助用户快速理解当前市场、跟踪自选股票、生成 AI 辅助分析报告。
- 系统不输出“买入答案”，而是输出**研究观察与交易计划**：给出数据依据、逻辑、风险、跟踪条件，决策由用户承担。
- 支持短线与长线两套不同推荐逻辑。
- 支持用户将股票标记为“已购入”，并记录买入价格、买入时间、持仓类型、来源推荐、卖出状态和复盘信息。
- 系统记录 AI 推荐历史，后续跟踪推荐表现，避免推荐结果只停留在一次性文本输出。
- LLM、行情数据源、用户偏好、风险偏好都需要持久化配置。

## 核心功能

1. 市场首页
   - 大盘指数、板块涨跌、热门行业、异动股票、市场情绪、资金流向、财经新闻、财报日历、宏观事件。
   - 提供 AI 今日市场摘要、风险提示和机会入口。

2. GitHub 登录与用户系统
   - 使用 GitHub OAuth 登录。
   - 保存用户偏好、风险等级、关注市场、默认推荐周期、默认模型配置。

3. 自选股
   - 添加、删除、分组、备注、重点关注。
   - 支持从自选股直接发起个股分析、短线推荐或长线研究。

4. 已购入持仓
   - 用户可以把股票标记为已购入。
   - 记录买入价、买入日期、数量、短线/长线类型、买入理由、来源推荐。
   - 查询时显示当前盈亏、推荐逻辑是否仍成立、是否触发卖出条件。

5. AI 分析中心
   - 可选择市场、板块、个股、自选股、已购入持仓、新闻情绪、技术面、基本面、宏观事件等模块。
   - 可选择是否推荐股票。
   - 推荐时先分析市场，再从候选池中输出 3 到 5 个标的。

6. 短线推荐
   - 输出短线交易计划：买入观察区间、止损位、止盈位、有效期、预计持有周期、失效条件。
   - 系统记录推荐过的短线标的。
   - 暂不强制做主动推送提醒，MVP 阶段用户查询时显示是否触发止盈、止损、过期或需要重新分析。

7. 长线推荐
   - 输出长期投资研究报告：基本面、行业空间、估值区间、财务质量、护城河、关键跟踪指标、财报观察点。
   - 重点是复盘和跟踪，不做高频买卖提示。

8. 推荐后追踪
   - 记录推荐日期、推荐价格、当前价格、最高涨幅、最大回撤、是否触发止盈/止损、是否过期、用户是否买入。
   - 形成 AI 推荐历史和表现统计。

9. 设置
   - LLM 连接配置：Provider、Base URL、API Key、Model、Temperature、Max Tokens、流式输出、测试连接。
   - 数据源配置：行情、财务、新闻、宏观数据源及刷新频率。
   - 策略模板、风险偏好、通知偏好。

## 技术方向

后端参考 `new-api` 项目的分层架构与工程组织方式：

- Go + Gin
- GORM
- 数据库：开发用 SQLite，生产用 MySQL（由宝塔托管，与 new-api 共存）；PostgreSQL 仅 GORM 层兼容，不主推
- Redis 缓存
- JWT / OAuth
- Controller / Service / Model 分层
- 配置、日志、任务、缓存、限流、AI 渠道管理等能力参考 `new-api`

前端不沿用 `new-api` 的 React 前端，改为独立 Vue 技术栈：

- Vue 3
- Vite
- TypeScript
- Pinia
- Vue Router
- ECharts
- Naive UI
- 内置 6 套可切换主题（亮/暗 × 蓝绿紫橙红），持久化到 localStorage；后续所有页面样式必须兼容全部主题（详见 [技术架构设计 §4.1](docs/ARCHITECTURE.md)）

兼容性判断：前端框架更换为 Vue 没有本质问题，只要后端暴露稳定的 REST API、SSE 或 WebSocket 接口即可。需要避免复用 `new-api/web/default` 中的 React 组件代码，但可以参考它的页面组织、接口封装、登录流程和设置页设计。

## 数据源策略

数据源是本项目工程量最大、最容易塌方的部分（每个市场每类数据的供应商、格式、限频、合规都不同），因此采取**先验证、后扩展**：

- **先用一个公开数据源、打通一个市场的行情，端到端跑通**（数据获取 → 标准化 → 缓存 → 展示 → 喂给 AI），证明能稳定拿到**真实、足够实时**的数据，再谈扩展更多市场/数据类型。当前选型为东方财富 + 新浪公开接口打底；Tushare Pro 分档按需启用（免费档与低 cost 档保留、高级档暂不实现），第一阶段不以 Tushare 为前置依赖，详见 [数据源选型](docs/DATA_SOURCES.md)。
- 后端设计**数据适配/标准化层**（不同数据源统一成内部标准结构），但 MVP 只实现一个源；新增源时只加适配器，不改上层。
- 所有数据必须携带**数据时间与来源**，AI 分析时明确告知数据时间范围，避免把旧数据当实时数据。

## 风险与合规

### 法律风险（重要）

在中国大陆，**向不特定对象推荐具体证券及买卖时点属于证券投资咨询业务，需证监会牌照**。无证“荐股”即使附带免责声明也可能违规。因此：

- 本项目定位为**个人自用研究工具**，谨慎对待公开多用户部署；公开前必须做合规评估。
- 全产品统一用“**研究观察 / 交易计划 / 候选标的**”表述，**禁止**出现“推荐买入”“稳赚”“目标收益”等确定性、诱导性措辞。
- 短线模块表述为“短线交易计划 / 买入观察计划”，强调条件与风险，不做主动买卖指令。

### 数据与模型风险

所有 AI 推荐与分析内容必须明确标注：

- 仅供研究参考，不构成投资建议。
- AI 分析可能出错，数据可能延迟或不完整；分析结论与输入数据快照绑定。
- 用户需自行承担投资决策风险。
