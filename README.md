# QuantVista

QuantVista 是一个完整全栈的 AI 股票研究与交易辅助平台。项目目标不是给出不可解释的“买入答案”，而是围绕市场数据、用户自选、AI 分析、短线/长线推荐、已购入持仓和推荐后追踪，形成可复盘、可持久化、可配置的股票研究工作台。

## 项目定位

- **部署形态：个人自用研究工具**。当前阶段定位为单人/极少数人自用，不做公开多用户商业部署。这一定位直接决定了合规风险等级和措辞口径（见“风险与合规”）。如未来需要公开部署，必须先补充合规评估。
- 面向个人投资研究场景，帮助用户快速理解当前市场、跟踪自选股票、生成 AI 辅助分析报告。
- 系统不输出“买入答案”，而是输出**研究观察与交易计划**：给出数据依据、逻辑、风险、跟踪条件，决策由用户承担。
- 支持短线与长线两套不同推荐逻辑。
- 支持用户将股票标记为“已购入”，并记录买入价格、买入时间、持仓类型、来源推荐、卖出状态和复盘信息。
- 系统记录 AI 推荐历史，后续跟踪推荐表现，避免推荐结果只停留在一次性文本输出。
- LLM、行情数据源、用户偏好、风险偏好都需要持久化配置。

## 核心功能（与当前实现一致，更新于 2026-07-03）

1. **市场首页与个股**
   - 大盘指数、涨幅榜、热门榜（成交额）、板块涨跌、涨跌家数情绪、两市资金流向、AI 今日观点（最新收盘日报摘要）。
   - 个股速查（行情 + 日 K + 腾讯免费估值快照），榜单行与速查可点入**个股详情页**（行情/日K/估值/技术评分五维 + 分析/问答/对比/提醒/逻辑卡/自选/建仓快捷动作）。
   - 财经新闻/财报日历/宏观事件依赖外部数据源，接入前不展示占位（见 [数据源选型 §7](docs/DATA_SOURCES.md)）。

2. **登录与用户系统**
   - 用户名密码 + GitHub OAuth 双登录；首次启动引导创建管理员；已登录用户可在设置页**绑定/解绑 GitHub**（绑定后同一 GitHub 直登本账号）。
   - 用户偏好：风险等级、默认市场、默认周期、默认推荐数量、推送总闸、候选池黑名单与流动性门槛、收盘日报开关。
   - 每用户 AI **次数配额**（一次手动分析/推荐/问答/点评计 1 次，后台任务不计次），token 用量作审计展示。

3. **自选股与已购入持仓**
   - 自选：分组、备注、重点置顶、研究阶段漏斗（发现→初筛→…→已买入/已放弃）、错过机会复盘。
   - 持仓：买入价/日期/数量/费税/短长线类型，实时盈亏（成本含费税），平仓结构化复盘；建仓前风险计算器 + 检查清单；组合总览（盈亏/分布/集中度/破近止损风控信号）。
   - 推荐↔持仓血缘：一键建仓记录来源推荐，回显推荐价 vs 实际买价。

4. **AI 分析中心**
   - 五模块（个股/全市场/板块/自选/持仓）分模块精确提示词 + 反方观点/失效条件/数据盲区；多角色 panel 模式（技术/动量/风险/反方四视角）；变化检测（与上次分析对比）；支持每模块自定义提示词模板。
   - 个股 AI 问答（多轮追问复用数据快照）、个股横向对比（含估值与评分、AI 点评）。

5. **短线/长线推荐（反编造硬约束）**
   - 候选池 = 自选 ∪ 涨幅榜 ∪ 活跃榜（黑名单/ST/流动性前置过滤），AI 只能从池内选，越池标的直接丢弃；落选标的给一句话理由。
   - 短线输出买入观察区间/止盈/止损/有效期/失效条件；长线输出逻辑/估值区间/跟踪指标/复盘周期。

6. **推荐后追踪**
   - 止盈/止损按当日 high/low 判盘中触达、按交易日历判过期；相对上证指数的超额收益 alpha；7/14/30 日节点收益；历史表现统计（样本量/胜率/均收益）。

7. **收盘日报（2026-07-03 新增）**
   - 交易日 15:35 后自动生成：**今日复盘**（市场+持仓当日表现+自选异动+命中提醒的 LLM 总结）+ **明日短线推荐**（买点区间/止盈/止损/有效期，落库可追踪）。
   - 每条推荐自动创建到价**卖点提醒**（有效期内命中即进待办并推送，过期自动暂停）；可推送摘要到 Server酱/Webhook；设置页偏好开关控制，默认关闭。

8. **提醒与待办**
   - 条件提醒：到价/异动/均线/突破/放量/振幅 × ≥/≤，每 15 分钟评估，命中明细带 已读/忽略 状态机；推送通道（Server酱/自定义 Webhook，加密存储、同日去重）。
   - 今日待办：聚合命中提醒、推荐/持仓复盘信号、逻辑卡到期，按优先级排序一键跳转。

9. **研究辅助**
   - 投资逻辑卡（核心逻辑/证据/失效条件/跟踪指标/复盘日期 + 一键体检）、投资笔记（时间线/标的关联）、股票技术面评分（五维 0-100）、模拟交易（虚拟 10 万、真实行情成交与费税）。

10. **设置与管理**
    - LLM 配置：多配置增删改查、默认配置、测试连接（OpenAI 兼容）、API Key 加密存储；AI 请求带连接池复用/瞬时错误重试/错误归类。
    - 数据：持仓 CSV 导入（模板/校验/错误行报告）、四类数据 CSV 导出。
    - 管理员后台：用户启停、次数配额调整与清零、GitHub 凭证与注册开关、市场数据维护（日线同步/日历回填/快照）、数据源同步日志。

## 技术方向

后端参考 `new-api` 项目的分层架构与工程组织方式：

- Go + Gin
- GORM
- 数据库：开发用 SQLite，生产用 MySQL（由宝塔托管，与 new-api 共存）；PostgreSQL 仅 GORM 层兼容，不主推
- Redis 缓存
- JWT / OAuth
- Controller / Service / Model 分层
- 配置、日志、任务、缓存、限流、AI 渠道管理等能力参考 `new-api`

前端不沿用 `new-api` 的 React 前端，改为独立 Vue 技术栈：

- Vue 3
- Vite
- TypeScript
- Pinia
- Vue Router
- ECharts
- Naive UI
- 内置 6 套可切换主题（亮/暗 × 蓝绿紫橙红，默认樱桃红·浅），持久化到 localStorage；后续所有页面样式必须兼容全部主题（详见 [技术架构设计 §4.1](docs/ARCHITECTURE.md)）

兼容性判断：前端框架更换为 Vue 没有本质问题，只要后端暴露稳定的 REST API、SSE 或 WebSocket 接口即可。需要避免复用 `new-api/web/default` 中的 React 组件代码，但可以参考它的页面组织、接口封装、登录流程和设置页设计。

## 数据源策略

数据源是本项目工程量最大、最容易塌方的部分（每个市场每类数据的供应商、格式、限频、合规都不同），因此采取**先验证、后扩展**：

- **先用一个公开数据源、打通一个市场的行情，端到端跑通**（数据获取 → 标准化 → 缓存 → 展示 → 喂给 AI），证明能稳定拿到**真实、足够实时**的数据，再谈扩展更多市场/数据类型。当前选型为东方财富 + 腾讯 + 新浪公开接口三源互备（**仅覆盖 A 股沪深**，美股/港股待接源后开放）；Tushare Pro 分档按需启用（免费档与低 cost 档保留、高级档暂不实现），第一阶段不以 Tushare 为前置依赖，详见 [数据源选型](docs/DATA_SOURCES.md)。
- 后端设计**数据适配/标准化层**（不同数据源统一成内部标准结构），但 MVP 只实现一个源；新增源时只加适配器，不改上层。
- 所有数据必须携带**数据时间与来源**，AI 分析时明确告知数据时间范围，避免把旧数据当实时数据。

## 风险与合规

### 法律风险（重要）

在中国大陆，**向不特定对象推荐具体证券及买卖时点属于证券投资咨询业务，需证监会牌照**。无证“荐股”即使附带免责声明也可能违规。因此：

- 本项目定位为**个人自用研究工具**，谨慎对待公开多用户部署；公开前必须做合规评估。
- 全产品统一用“**研究观察 / 交易计划 / 候选标的**”表述，**禁止**出现“推荐买入”“稳赚”“目标收益”等确定性、诱导性措辞。
- 短线模块表述为“短线交易计划 / 买入观察计划”，强调条件与风险，不做主动买卖指令。

### 数据与模型风险

所有 AI 推荐与分析内容必须明确标注：

- 仅供研究参考，不构成投资建议。
- AI 分析可能出错，数据可能延迟或不完整；分析结论与输入数据快照绑定。
- 用户需自行承担投资决策风险。
