# 实施路线图

## 阶段 0：项目初始化

目标：建立可运行的全栈骨架。

任务：

- 初始化 Go 后端工程。
- 初始化 Vue 3 + Vite + TypeScript 前端工程。
- 建立 `docs/`、`server/`、`web/`、`deploy/` 等目录。
- 配置 Docker Compose。
- 配置数据库连接。
- 配置 Redis，允许本地可选。
- 建立基础日志和错误返回格式。

验收：

- 后端健康检查接口可访问。
- 前端开发服务器可启动。
- 前端能调用后端健康检查接口。

## 阶段 1：用户与设置

目标：完成登录、用户持久化和 LLM 配置。

任务：

- GitHub OAuth 登录。
- JWT 鉴权。
- 用户表和用户偏好表。
- LLM 配置增删改查。
- LLM API Key 加密保存。
- LLM 测试连接。
- 用户偏好设置：风险等级、默认市场、默认周期、默认推荐数量。

验收：

- 用户可以使用 GitHub 登录。
- 登录后可以保存 LLM 配置。
- 前端不会显示明文 API Key。
- 测试连接结果可以展示成功或失败原因。

## 阶段 2：市场数据与首页

目标：让用户进入系统后可以看到市场状态。

任务：

- 股票基础信息表。
- 行情快照表。
- 市场快照表。
- 市场首页接口。
- 首页图表和榜单。
- 数据更新时间展示。
- 数据源异常提示。

验收：

- 首页可以展示指数、板块、热门股票和市场摘要。
- 行情数据有缓存。
- 数据更新时间清晰可见。

## 阶段 3：自选股与已购入持仓

目标：完成用户个人股票管理。

任务：

- 自选股分组。
- 自选股添加、删除、编辑备注。
- 标记重点关注。
- 已购入持仓增删改查。
- 从自选股创建持仓。
- 记录买入价、买入日期、数量、短线/长线类型。
- 持仓列表显示当前盈亏。
- 支持标记已卖出和填写复盘。

验收：

- 自选股和持仓数据按用户隔离。
- 用户可以将股票标记为已购入。
- 持仓页面可以展示短线和长线分类。

## 阶段 4：AI 分析中心

目标：完成可配置的 AI 分析。

任务：

- 分析请求接口。
- 市场、板块、个股、自选股、持仓等分析模块。
- 组装数据上下文。
- 调用用户 LLM 配置。
- 结构化输出校验。
- 保存分析历史。
- AI 调用日志和 token 统计。
- 前端分析中心页面。

验收：

- 用户可以选择模块发起 AI 分析。
- 分析结果持久化。
- 分析历史可查询。
- 失败时能看到明确错误。

## 阶段 5：短线/长线推荐

目标：实现核心推荐能力。

任务：

- 策略模板。
- 候选池筛选。
- 推荐生成接口。
- 短线推荐输出买入观察区间、止盈、止损、有效期、失效条件。
- 长线推荐输出基本面逻辑、估值区间、关键指标、复盘周期。
- 推荐结果保存。
- 从推荐一键加入已购入持仓。

验收：

- 用户可以选择短线或长线推荐。
- 系统推荐 3 到 5 个标的。
- 推荐结果必须包含理由、风险、数据依据和免责声明。
- 推荐股票来自候选池，不允许 AI 无依据编造。

## 阶段 6：推荐追踪与短线卖出提示

目标：补齐推荐后的闭环。

任务：

- 推荐追踪表。
- 定时更新当前价、最高价、最低价。
- 计算当前收益率、最大涨幅、最大回撤。
- 判断止盈、止损、过期、需要重新分析。
- 持仓页展示短线状态提示。
- 推荐历史页展示表现统计。

验收：

- 用户查看短线推荐或短线持仓时，能看到是否该复盘、是否触发止盈/止损、是否过期。
- 系统能展示 AI 推荐历史表现。
- 不做主动推送也不影响查询提示。

## 阶段 7：增强功能

目标：提升产品完整度和可信度。

任务：

- 股票评分系统。
- 新闻情绪分析。
- 财务数据详情。
- 模拟交易。
- 主动提醒。
- 回测模块。
- 管理员后台。
- 多数据源配置。
- Prompt 模板管理。

验收：

- 推荐表现可以量化。
- 用户可以复盘 AI 推荐质量。
- 管理员可以维护系统级数据源和模型配置。

## 技术风险

### 数据源风险

股票行情和财务数据质量直接影响 AI 分析质量。必须显示数据更新时间，并在数据异常时阻止或弱化推荐。

### LLM 幻觉风险

AI 必须基于候选池和输入数据分析。推荐输出需要结构化校验，并且保存输入数据快照。

### 合规风险

产品必须使用“研究参考”和“交易计划”表述，避免包装成确定收益或强投资建议。

### 成本风险

AI 调用需要缓存、频率限制和 token 日志。首页不能每次刷新都调用 LLM。
