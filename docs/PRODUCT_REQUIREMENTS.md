# 产品需求文档

## 1. 产品目标

QuantVista 要做的是一个带数据持久化的 AI 股票研究与交易辅助平台。用户可以查看实时市场信息，管理自选股和已购入持仓，调用自己配置的 LLM 对市场、板块、个股和持仓进行分析，并根据短线或长线目标生成可跟踪的推荐结果。

核心闭环：

1. 用户查看当前市场状态。
2. 用户添加自选股或已购入持仓。
3. 用户选择分析模块和投资周期。
4. 系统汇总行情、财务、新闻、宏观和用户偏好数据。
5. AI 生成结构化分析报告。
6. 如果用户勾选推荐，系统输出 3 到 5 个候选股票。
7. 系统持久化推荐结果并持续跟踪表现。
8. 用户再次查询时，系统提示短线是否触发卖出条件，长线是否需要复盘。

## 2. 用户角色

### 普通用户

- 使用 GitHub 登录。
- 查看市场首页。
- 管理自选股。
- 管理已购入持仓。
- 配置自己的 LLM。
- 发起 AI 分析和推荐。
- 查看推荐历史、分析历史和持仓状态。

### 管理员

- 管理系统级数据源。
- 管理默认 LLM 渠道。
- 查看系统调用日志。
- 配置全局风控提示和免责声明。
- 管理用户、任务、缓存和异常日志。

## 3. 功能模块

### 3.1 市场首页

首页用于快速回答“当前市场怎么样”。

功能内容：

- 大盘指数：按市场展示主要指数。
- 板块涨跌：行业、概念、主题排行。
- 热门股票：涨幅、跌幅、成交额、换手率、异动排行。
- 市场情绪：涨跌家数、涨停跌停、波动率、新闻情绪。
- 资金流向：行业资金、ETF 资金、主力资金、北向资金，视数据源能力实现。
- 新闻事件：财经新闻、公司公告、财报日历、宏观事件。
- AI 今日观点：自动生成市场摘要、机会方向和风险提示。

页面要求：

- 清晰显示数据更新时间。
- 数据源异常或延迟时必须提示。
- 首页第一屏直接展示可操作的市场看板，不做营销式落地页。

### 3.2 GitHub 登录与用户系统

功能内容：

- GitHub OAuth 登录。
- 自动创建用户。
- 保存用户头像、昵称、邮箱、GitHub ID。
- 支持用户风险偏好设置。
- 支持用户默认市场、默认推荐周期、默认 LLM 配置。

用户偏好：

- 风险等级：保守、平衡、激进。
- 默认市场：A 股、美股、港股，可扩展。
- 投资周期偏好：短线、中线、长线。
- 默认推荐数量：3 到 5。
- 是否启用通知。
- 是否启用模拟交易。

### 3.3 自选股

功能内容：

- 添加股票到自选。
- 删除自选股。
- 自选股分组。
- 设置重点关注。
- 添加关注原因和备注。
- 显示实时行情、涨跌幅、成交量、所属行业。
- 显示技术指标和基本面摘要。
- 从自选股发起 AI 分析。
- 从自选股标记为已购入。

自选股不是持仓。用户可以只关注，也可以进一步创建已购入记录。

### 3.4 已购入持仓

“已购入”建议做成独立持仓记录，而不是简单标签。原因是后续需要跟踪盈亏、卖出状态、复盘和推荐质量。

功能内容：

- 手动添加已购入股票。
- 从 AI 推荐结果一键加入已购入。
- 从自选股标记为已购入。
- 记录买入价格、买入日期、数量、持仓类型。
- 记录买入理由和备注。
- 展示当前价格、当前盈亏、持仓天数。
- 展示来源推荐和 AI 当时的分析依据。
- 支持标记已卖出。
- 支持卖出复盘。

持仓类型：

- 短线：关注止盈、止损、有效期、卖出触发条件。
- 长线：关注基本面变化、估值变化、财报复盘和长期逻辑是否破坏。

### 3.5 AI 分析中心

用户可选择分析范围：

- 当前市场
- 指定市场
- 指定板块
- 指定个股
- 自选股
- 已购入持仓
- 新闻情绪
- 技术面
- 基本面
- 宏观事件

用户可选择分析目标：

- 只分析，不推荐
- 分析并推荐股票
- 生成短线计划
- 生成长线研究
- 复盘历史推荐
- 复盘已购入持仓

AI 输出必须结构化：

- `summary`：结论摘要
- `market_context`：市场环境
- `data_points`：关键数据依据
- `opportunities`：机会方向
- `risks`：风险因素
- `recommendations`：推荐列表
- `confidence`：置信度
- `next_watch_points`：后续观察点
- `disclaimer`：免责声明

### 3.6 股票推荐

推荐数量：

- 默认 3 个。
- 用户可选择 3 到 5 个。

推荐前置筛选：

- 流动性足够。
- 成交额足够。
- 排除停牌、退市风险、明显异常股票。
- 按市场、行业、风险偏好、投资周期过滤。
- 先由规则和数据筛出候选池，再交给 AI 做解释和排序。

推荐结果字段：

- 股票代码
- 股票名称
- 市场
- 推荐类型：短线 / 长线
- 推荐价格或分析基准价
- 推荐理由
- 数据依据
- 风险点
- 置信度
- 有效期
- 触发条件
- 失效条件

### 3.7 短线推荐与卖出提示

短线推荐不建议在产品文案中写成“AI 直接买”，应表述为“短线交易计划”或“买入观察计划”。系统可以提供明确的买卖条件，但最终决策由用户承担。

短线推荐输出：

- 买入观察区间
- 推荐基准价
- 止损位
- 止盈位
- 预计持有周期
- 推荐有效期
- 卖出触发条件
- 失效条件
- 风险等级

短线跟踪状态：

- `watching`：观察中
- `active`：用户已购入，计划生效
- `take_profit_triggered`：触发止盈
- `stop_loss_triggered`：触发止损
- `expired`：超过有效期
- `needs_review`：需要重新分析
- `closed`：用户已卖出

MVP 阶段不做主动提醒。用户进入页面时展示：

- 是否已触发止盈。
- 是否已触发止损。
- 是否超过推荐有效期。
- 是否需要重新分析。

### 3.8 长线推荐与持仓复盘

长线推荐关注公司质量和长期逻辑，不应使用短线止盈止损作为核心。

长线推荐输出：

- 公司基本面摘要
- 行业空间
- 竞争优势
- 财务质量
- 估值区间
- 建仓观察区间
- 关键跟踪指标
- 财报观察点
- 长期风险
- 建议复盘周期

长线复盘触发：

- 财报发布后
- 估值显著变化
- 行业政策变化
- 公司基本面恶化
- 长期逻辑被破坏

### 3.9 推荐后追踪

系统必须记录 AI 推荐过什么，并持续计算表现。

追踪指标：

- 推荐日期
- 推荐基准价
- 当前价格
- 最高价
- 最低价
- 当前收益率
- 最高收益率
- 最大回撤
- 是否达到止盈
- 是否触发止损
- 是否过期
- 用户是否买入
- 用户是否卖出

统计指标：

- 短线推荐成功率
- 平均收益率
- 平均持有天数
- 最大回撤
- 触发止损比例
- 长线推荐表现
- AI 推荐与用户实际买入表现对比

### 3.10 策略模板

内置策略：

- 短线动量
- 突破买入
- 回调买入
- 低估值长线
- 高股息
- 成长股
- 防御型
- 财报超预期
- 行业轮动

策略模板用于约束候选池和 Prompt，避免 AI 从全市场无依据推荐。

### 3.11 股票评分系统

每只股票可以生成多维评分：

- 趋势分
- 估值分
- 成长分
- 财务健康分
- 情绪分
- 风险分
- 综合分

评分由规则和数据生成，AI 用于解释评分、发现风险和排序，不直接替代数据判断。

### 3.12 LLM 设置

用户级 LLM 配置：

- Provider
- Base URL
- API Key
- Model
- Temperature
- Max Tokens
- 是否流式输出
- 测试连接
- 默认分析模型
- 默认推荐模型

安全要求：

- API Key 后端加密保存。
- 前端不回显明文 API Key。
- 支持删除和替换。
- 调用日志记录 token 消耗、耗时、错误信息。

### 3.13 数据源设置

数据源类型：

- 行情数据
- 财务数据
- 新闻数据
- 宏观数据
- 公司公告

要求：

- 显示数据更新时间。
- 显示数据源健康状态。
- 失败时写入日志。
- AI 分析必须知道数据时间范围，避免把旧数据当成实时数据。

### 3.14 分析历史

功能内容：

- 保存每次 AI 分析报告。
- 支持按日期、股票、市场、模块、推荐类型筛选。
- 支持重新运行分析。
- 支持从历史分析创建推荐或持仓。

### 3.15 模拟交易

建议作为第二阶段功能。

功能内容：

- 模拟买入。
- 模拟卖出。
- 模拟持仓收益。
- 对比 AI 推荐表现和用户模拟操作表现。

## 4. 非功能需求

### 性能

- 行情数据走缓存。
- AI 分析结果可按参数缓存。
- 首页接口应避免每次加载都触发 LLM。

### 安全

- OAuth 登录。
- JWT 鉴权。
- 用户数据隔离。
- API Key 加密保存。
- 防止 SSRF，尤其是用户可配置 Base URL 和数据源 URL 时。

### 可观测性

- 请求日志。
- AI 调用日志。
- 数据源同步日志。
- 定时任务日志。
- 错误日志。

### 合规

所有 AI 推荐和分析页面必须展示风险声明：

> 本内容仅供研究参考，不构成投资建议。AI 可能出错，数据可能延迟或不完整，投资决策需由用户自行承担风险。
