# 编译推送步骤

> 与 new-api 一致：本地构建多架构镜像 → 推送到 Docker Hub → 宝塔容器编排更新版本重启。

## 1. 改版本 tag

每次发布换一个 tag（日期 + 序号），例如 `custom-20260629-01`。
同时改两处保持一致：

- `deploy/.env` 里的 `IMAGE`
- 下面构建命令里的 tag

## 2. 编译（在项目根目录执行）

```bash
docker buildx build --platform linux/amd64 -t raoczh/quantvista:custom-20260629-01 --load .
```

## 3. 推送

```bash
docker push raoczh/quantvista:custom-20260629-01
```

## 4. 部署（宝塔）

1. 进入「容器编排」→ 找到 quantvista 编排。
2. 把 compose 里的 image tag 改成新 tag（或编排引用 `${IMAGE}`，则改 env 即可）。
3. 重新部署 / 重启。
4. 应用启动时会自动连 MySQL 并执行 AutoMigrate 建表/加字段，无需手动改表结构。
5. 等健康检查变绿：`http://<host>:3002/api/status` 返回 `"success": true`。

## 5. 释放 docker 缓存（可选）

```bash
docker system prune -a --volumes
```
